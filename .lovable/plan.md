
# מערכת ריבוי דפי נחיתה (Multi-Landing Pages)

## מטרה
להפוך את האתר הקיים מ"אתר אחד" ל"מערכת לניהול מאות דפי נחיתה" — כל דף עם:
- נתיב ייחודי משלו (לדוגמה: `mysite.lovable.app/lp/makeup-tlv`)
- וובהוק זאפייר משלו (Zapier נפרד לכל דף)
- תוכן משלו (כותרות, תמונות, צבעים, טפסים) שנבנה איתי

> הערה על סאב-דומיינים: Lovable לא תומך בסאב-דומיינים דינמיים. אז נשתמש בנתיב (`/lp/<slug>`). אם תרצה בעתיד `client.yourdomain.com` — זה ידרוש דומיין משלך + Cloudflare Workers (שלב מתקדם, לא עכשיו).

---

## מבנה המערכת

### 1. מסד נתונים (טבלה חדשה: `landing_pages`)
שדות עיקריים:
- `slug` — מזהה ייחודי לנתיב (`makeup-tlv`)
- `title` — כותרת הדף
- `subtitle` — תיבת "בשיתוף..."
- `hero_heading` — הכותרת הראשית
- `cta_text` — טקסט כפתור
- `branch` — סניף ברירת מחדל לליד
- `zapier_webhook_url` — וובהוק זאפייר ייחודי לדף הזה
- `images` — מערך JSON של תמונות לקולאז'
- `theme` — JSON לצבעים/עיצוב בסיסי
- `is_published` — האם הדף פעיל
- `extra_content` — JSON לתוכן חופשי (סקציות נוספות)

הטבלה `leads` תקבל עמודה חדשה `landing_page_id` כדי לדעת מאיזה דף הגיע הליד.

### 2. נתיבים חדשים
- `/lp/$slug` — מציג דף נחיתה לפי ה-slug, שולח לידים לזאפ של אותו דף
- `/admin` — רשימת כל הדפים + יצירה/עריכה/מחיקה
- `/admin/page/$slug` — עריכת דף ספציפי (תוכן + וובהוק + תמונות)
- `/` — הדף הראשי הקיים יישאר כדף ברירת מחדל (או נהפוך אותו ל"אחד מהדפים" — תחליט)

### 3. שליחת לידים
פונקציית שרת `submitLead` תקבל `landing_page_id`, תשמור את הליד במסד, ותעביר ל-`zapier_webhook_url` של הדף הספציפי (לא הגלובלי).

### 4. אבטחה
- העריכה ב-`/admin` כבר מוגנת בסיסמה (כפי שקיים)
- RLS על `landing_pages`: קריאה פומבית רק לדפים `is_published=true`, כתיבה רק דרך admin

---

## תיעוד וזיכרון
1. אפתח קובץ **Skill** ב-`.agents/skills/landing-pages-system/SKILL.md` עם הסבר מלא איך המערכת עובדת (סכמה, נתיבים, איך מוסיפים דף, איך מחברים זאפ).
2. אעדכן את **mem://index.md** עם כללי הליבה של הפרויקט (מערכת רב-דפית, איך בונים דף חדש).

---

## שלבים לביצוע (לפי הסדר)
1. **מיגרציית מסד נתונים** — יצירת `landing_pages` + עמודה ב-`leads` + RLS + GRANTs
2. **קומפוננטת תבנית דף נחיתה** משותפת (`LandingPageView`) שמקבלת את התוכן מהמסד
3. **נתיב `/lp/$slug`** — טוען את הדף מהמסד
4. **שדרוג `/admin`**: רשימה + יצירה + עריכה של דפים (כולל וובהוק לכל אחד)
5. **עדכון `submitLead`** לשימוש בוובהוק הספציפי של הדף
6. **migration נתונים**: אעביר את הדף הקיים (קורס מאפרת) להיות "הדף הראשון" במערכת
7. **כתיבת Skill + Memory** לתיעוד

---

## מה אצטרך ממך אחרי האישור
- שם slug לדף הראשון (לדוגמה `makeup-pro`)
- האם הדף הראשי `/` נשאר הדף הקיים, או הופך לעמוד "בחר דף נחיתה"/landing index?
- בעריכת תוכן — האם מספיק בשלב ראשון: כותרת, תיבה, תמונות, וובהוק, סניף? (תוספות מתקדמות כמו סקציות חופשיות נוכל להוסיף בהמשך)

לאחר אישור הפלאן — אתחיל מהמיגרציה.
